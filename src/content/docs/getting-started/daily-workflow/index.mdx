# Daily workflow

A simple daily rhythm works well for small product teams.

## Suggested routine
- Review the active sprint scope.
- Update stories as work moves forward.
- Keep ReleaseHub versions honest about what is really shipping.
- Refresh DocHub drafts when a story changes user-facing behavior.
